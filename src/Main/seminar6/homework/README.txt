Какие могут быть User ы  - админы, гостевые, пользователи.
Считаю, что сохранять мы будем всех пользователей, а построение отчётов можно делать не всем пользователям

Так как в домашнем задании у нас нет никаких видов пользователей кроме обычных, тогда я считаю, что мы будем сохранять
всех пользователей, то есть метод save будет в классе User, а метод Report я перемещаю в интерфейс Reportable, к
которому мы сможем давать доступ разным пользователям

Для закрытия кода я создаю класс ProgrammRunner с приватным методом работы программы, который будет у нас запускать
программу

P.S. По информации из интернета Persister - это метод, который собирает информацию.
На примере наших методов:
При сохранение пользователя(метод Save):
Вызываемый метод Save обращается в Persister, передаёт ему пользователя для сохранения, метод saveUser в Persister
передаёт данные для сохранения(в нашем случае пишет, что пользователь сохранён и его имя).

По аналогии рабоает с построением отчёта методом Report

// # НОВЫЙ

По принципу единственной ответственности класс User является только классом сущности.
За сохраниение отвечает класс Persister, который имеет методы от интерфейса Persisterable.
За составление отчёта отвечает класс Reporter, который имеет методы от интерфейса Reportable.
